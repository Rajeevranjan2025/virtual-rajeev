Q: What is Virtual Rajeev?
A: Virtual Rajeev is a local AI assistant built as a learning-focused project.
It runs on a local machine using a Large Language Model (LLM) via Ollama and
uses Retrieval Augmented Generation (RAG) to answer questions from stored data.

--------------------------------------------------

Q: What was the main goal of building this project?
A: The main goal was to understand how real-world AI assistants are built and
controlled, focusing on system design, memory, retrieval, hallucination control,
privacy, and deployment concepts rather than only programming syntax.

--------------------------------------------------

Q: What is an LLM and how is it used in this project?
A: An LLM (Large Language Model) is a model that generates text responses.
In this project, a pre-trained open-source LLM is used locally via Ollama.
The model does not know personal data unless it is explicitly provided as context.

--------------------------------------------------

Q: What is Retrieval Augmented Generation (RAG)?
A: RAG is a technique where the AI retrieves relevant information from stored
knowledge before generating a response. This ensures answers are based on real
data instead of guessing, which reduces hallucinations.

--------------------------------------------------

Q: How does Virtual Rajeev store knowledge?
A: Knowledge is stored in structured text files (such as resume details,
skills, work experience, and learning documentation). These files are converted
into embeddings and indexed using FAISS for fast semantic search.

--------------------------------------------------

Q: What is FAISS and why is it used?
A: FAISS is a vector database library used for fast similarity search.
It helps find the most relevant information based on meaning rather than keywords.

--------------------------------------------------

Q: How does the AI decide which file to answer from?
A: When a user asks a question, the question is converted into an embedding.
FAISS finds the most similar stored embeddings, and the corresponding text
is provided to the AI as context.

--------------------------------------------------

Q: How is hallucination prevented in this project?
A: Hallucination is prevented using strict system prompts that instruct the AI
to answer only from the retrieved context. If information is not present, the
AI explicitly responds that the information is not mentioned.

--------------------------------------------------

Q: How is user privacy handled?
A: Rule-based checks are implemented to block sensitive personal data such as
phone numbers and emails. The AI is designed to protect private information.

Q: Why does the Telegram bot stop when the laptop is off?
A: Because the AI assistant and Telegram bot run locally.
When the local machine is turned off, the services are unavailable.
This is a conscious design choice for privacy and cost control.
